FASTAPI BACKEND CONTROL FLOW SUMMARY
====================================================================

Request lifecycle (protected routes):
1) Client sends HTTP request with Authorization: Bearer <token>.
2) FastAPI resolves Depends(get_auth_context).
3) Token is validated via Supabase Auth.
4) Route executes using AuthContext(user_id, access_token).
5) DB helper receives access_token to enforce RLS in PostgREST calls.
6) Response is returned as JSON or mapped to HTTPException.

Ingest route chain:
- clone_or_pull -> load_repo_documents -> chunk_documents -> ingest_to_pinecone -> save_repo

Query route chain:
- load_vectorstore(user-scoped namespace) -> build_rag_chain -> answer + source chunks

Repos route chain:
- list user rows in user_repos OR delete one user-owned row
